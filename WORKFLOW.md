# NextSpace 플러그인 개발 및 협업 워크플로

이 프로젝트는 **개발 효율성**과 **배포 안정성**을 동시에 잡기 위해 **하이브리드 로더(Hybrid Loader)** 시스템을 채택하고 있습니다.

---

## 🚀 0. 핵심 요약

| 모드 | 실행 명령어 | 동작 방식 | 주요 용도 |
|------|------------|----------|-----------|
| **개발 모드** | `py serve.py` | `localhost:8080`에서 실시간 코드 로드 | UI 수정 및 기능 개발 |
| **고정 모드** | `npm run build` | 로컬 번들(`dist/`) 생성 및 활용 | 개발 서버 없는 환경에서 확인 |
| **배포 모드** | `py deploy.py` | 단일 파일 번들링 후 서버 업로드 | 실제 운영 환경 적용 |

---

## 🏗️ 1. 프로젝트 구조 및 파일 역할

```
plugin/
  index.js          ← [메인 입구] 환경에 따라 개발자 도구 또는 번들을 선택해서 로드
  core.js           ← [핵심 로직] UI 렌더링 및 이벤트 처리가 모두 들어있는 본체 (1순위 수정 대상)
  plugin.js         ← [개발 엔트리] 동적 import를 지원하는 개발 전용 파일
  deployEntry.js    ← [배포 엔트리] 번들링을 위한 정적 import 기반 파일
  config/
    menuConfig.js   ← 대시보드 메뉴 및 탭 구조 정의
  components/
    pages/          ← 각 탭별 독립적인 화면 컴포넌트들
    layout/         ← 사이드바, 헤더 등 공통 레이아웃
```

---

## 💻 2. 개발 단계 (수정 및 확인)

1. **서버 실행**: `py serve.py`를 실행하여 8080 포트를 엽니다.
2. **코드 수정**: `plugin/config/menuConfig.js`나 `plugin/components/pages/`의 파일들을 수정합니다.
3. **실시간 확인**: NextSpace Navigator에서 플러그인을 켭니다. 수정 사항이 즉시 반영됩니다.
   - *팁: 반영이 안 되면 플러그인을 껐다 켜거나 브라우저 캐시 새로고침을 해주세요.*

---

## 🛠️ 3. 마무리 단계 (빌드 및 배포)

1. **로컬 고정 (선택)**:
   - `npm run build`를 실행하면 현재 코드가 `plugin/dist/index.bundle.js`로 합쳐집니다.
   - 이제 `serve.py`를 꺼도 마지막으로 작업한 내용이 플러그인에 나옵니다.

2. **최종 배포 (필수)**:
   - `py deploy.py`를 실행합니다.
   - 내부적으로 코드를 압축하고 번들링하여 NextSpace 서버로 안전하게 업로드합니다.

---

## 🤝 4. 협업 시 주의사항

- **공통 로직**: UI의 동작이나 레이아웃 수정은 `plugin/core.js`에서 진행하세요.
- **새 탭 추가**: 
  1. `menuConfig.js`에 탭 정보 추가
  2. `pages/` 폴더에 새 컴포넌트 파일 생성
  3. `plugin.js`와 `deployEntry.js` 양쪽에 새 컴포넌트를 등록해줘야 합니다.
- **의존성**: 새로운 라이브러리를 쓰려면 `package.json`에 추가하고 팀원들에게 `npm install`을 알리세요.
